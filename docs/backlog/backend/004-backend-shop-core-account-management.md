# (backlog) 계정 관리 — 비밀번호 재설정/변경, 회원 정보 수정/탈퇴

> 상태: backlog (미착수)
> 영역: shop-core (backend / member, 일부 view)
> 출처: Task 007 범위 밖 — "비밀번호 재설정·변경, 회원 정보 수정/탈퇴"

## 배경 / 동기
- Task 007은 회원가입 + 로그인 진입까지만 구현. 가입 이후 계정 라이프사이클 관리가 미구현.

## 범위 (할 것)
- 비밀번호 변경(로그인 상태, 현재 비밀번호 확인 + 신규 비밀번호 정책 검증).
- 비밀번호 재설정(비로그인, 이메일 토큰 기반) — 재설정 토큰 발급/검증(만료·1회용), 메일 발송은 notification 이벤트 연계 여부 결정.
- 회원 정보 수정(name/phone 등), 탈퇴(소프트 삭제 정책 결정 — `users` 스키마 영향 시 Flyway V_).

## 범위 밖 / 주의
- 비밀번호 원문 저장/로그/응답 금지(007 제약 계승), BCrypt 유지.
- 재설정 토큰은 Redis(`shopcore:auth:*`) 또는 별도 저장 — namespace 분리 유지.
- 탈퇴 시 주문/리뷰 등 연관 데이터 정책(보존/익명화)은 해당 도메인 Task와 정합.

## 선행 의존
- Task 006(JWT), 007(member) 완료. 재설정 메일은 notification 발송 Task와 연계 가능.
